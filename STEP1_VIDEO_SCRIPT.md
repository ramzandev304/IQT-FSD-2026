# Step 1: Video Submission Script (2-3 minutes)

Note: This is a starting draft built from what you told me (7 years of full-stack
experience). Personalize the bracketed [placeholders] with your real details before
recording — an interviewer may follow up on specifics, so make sure everything you
say is something you can back up.

---

**Intro (~15s)**

"Hi, I'm [your name], a full-stack developer with 7 years of experience building
web applications end to end — from the UI down to the database."

**1. Why should we hire you for this role? (~30-40s)**

"I bring 7 years of hands-on full-stack experience, which means I can own a
feature from a Figma design all the way to a deployed, tested API — I don't need
to hand off between a 'frontend person' and a 'backend person.' [Add 1-2
specifics: e.g., "I've worked in fast-moving startups where I had to move
quickly without sacrificing code quality," or "I've maintained production
systems, so I think about performance, security, and maintainability, not just
getting a feature working."] I also communicate clearly with designers, QA, and
PMs, which matters for a remote role like this one where async communication is
everything."

**2. Tell us about a web application you built and your specific contributions (~40-50s)**

"One project I can walk through is this Task Manager app I built as part of
this application — it's a full CRUD task management tool with a React
front-end and a Node/Express REST API backed by PostgreSQL. I designed the
database schema, built the REST endpoints for creating, reading, updating, and
deleting tasks, and built the React UI including add/edit/delete and
mark-complete functionality. I also integrated a public currency exchange API
to demonstrate third-party API integration. [If you have a real prior project,
swap this in and describe: what the app did, your specific role, and one
concrete technical decision you made and why.]"

**3. Which front-end and back-end technologies are you most comfortable with and why? (~30-40s)**

"On the front end, I'm most comfortable with React — I like its component
model and the ecosystem around it (React Query, hooks, etc.) for building
maintainable UIs. On the back end, I favor Node.js with Express because it
lets me share language and tooling across the stack, which speeds up
development. For data, I default to PostgreSQL when the data is relational and
consistency matters, and I'll reach for MongoDB when the schema needs to be
more flexible. [Adjust this to reflect your actual strongest stack if
different — e.g., .NET, Laravel, Vue, etc.]"

**4. Describe a technical challenge you faced and how you solved it (~30-40s)**

"[Replace with a real example. Structure: (1) what the problem was, (2) why it
was hard, (3) what you tried, (4) what actually fixed it, (5) what you learned.
Example shape: 'On a past project we had an API endpoint that was timing out
under load. I profiled it, found an N+1 query problem where we were hitting
the database once per row instead of batching, fixed it with a single joined
query and added an index, and cut response time from 4 seconds to under
200ms. It taught me to always check query patterns before assuming I need more
infrastructure.']"

**Closing (~10s)**

"Thanks for considering my application — I'm excited about the opportunity and
happy to walk through any part of the code or answer more questions."

---

**Recording tips:**
- No edits allowed per the instructions, so do 1-2 practice run-throughs first.
- Keep it to 2-3 minutes total — roughly 60-70 words per 30 seconds at a
  natural speaking pace.
- Speak to the camera like you're explaining it to a colleague, not reading
  word-for-word — use this as talking points, not a script to recite.
